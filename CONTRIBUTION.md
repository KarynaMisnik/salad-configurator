# Contributing Guide

## 🔀 Branching

- Always create a new branch for your work:

  ```bash
  git checkout -b feature/your-feature-name
  ```

- Use clear names:
  - `feature/...` for new features
  - `fix/...` for bug fixes

---

## 💬 Commits & Comments

- Write clear commit messages:

  ```
  Add login validation
  Fix issue with API request
  ```

- Keep commits small and focused
- Comment your code when logic is not obvious

---

## 📥 Pull Requests (PR)

- Push your branch:

  ```bash
  git push origin your-branch-name
  ```

- Open a Pull Request (PR) to `main`
- Describe:
  - What you did
  - Why it’s needed

---

## 🚫 Merging Rules

- ❗ Do NOT merge your own PR without approval
- Wait for review from repo owner
- Discuss changes if needed

---

## ✅ Review Process

- At least **1 approval required** before merge
- Fix requested changes before merging

---

## 🧠 General Good Practices

- Pull latest changes before starting:

  ```bash
  git pull origin main
  ```

- Keep code clean and readable
- Ask if unsure 👍
